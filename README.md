## Описание

Лабораторная работа посвящена задаче **обнаружения объектов** (object detection) на изображениях.

Исследуются современные детекторы семейства **Ultralytics YOLOv11** (n / s / m), трансформерная модель **RT-DETR-L**, а также собственный SSD-подобный детектор на PyTorch (MobileNetV2 + FPN + SSD-head). Все модели обучаются и сравниваются на игровом датасете с пятью классами объектов.

## Что реализовано

- Скачан и подготовлен датасет **YOLOv5 Game Dataset for Darknet Framework** с Kaggle
- Автоматическое разбиение на train / val / test в пропорции 70 / 20 / 10 и генерация `dataset.yaml`
- Визуализация случайных примеров с ground-truth bbox-ами
- Обучены бейзлайн-модели: `YOLO11n`, `YOLO11s`, `YOLO11m`, `RT-DETR-L`
- Проверены три гипотезы по улучшению бейзлайна:
  - умеренная аугментация (mosaic + HSV + геометрия);
  - увеличение входного разрешения с 640 до 800;
  - Cosine LR + warmup + больше эпох
- Собрана комбинированная улучшенная конфигурация `improved_final`
- Реализован собственный SSD-подобный детектор с нуля на PyTorch:
  - MobileNetV2 (предобучен на ImageNet) + FPN + SSD-head;
  - свой loss (CrossEntropy + hard negative mining 3:1, Smooth L1 для боксов);
  - своя реализация `mAP@50`, `Precision` и `Recall` (последние — в точке max F1, усреднение по классам);
  - декодирование через NMS
- Обучены и сравнены две версии кастомной модели: базовая и улучшенная
- Построены сравнительные графики и финальная сводка

## Вывод

Лучшей моделью стала **YOLO11s с увеличенным входным разрешением `imgsz=800`** (эксперимент `h2_larger_img`): `mAP@50 = 0.9479`, `mAP@50-95 = 0.5879`, `Recall = 0.9133`.

Основные наблюдения:

- `YOLO11s` — лучший бейзлайн и по `mAP@50` (0.9248), и по строгой `mAP@50-95` (0.5712), опережает в том числе `RT-DETR-L` (0.8990 / 0.5531);
- наибольший прирост качества обеспечивает увеличение входного разрешения (`640 → 800`), особенно для мелких объектов (`coconut`, `fluff`);
- умеренная аугментация и Cosine LR по отдельности дают лишь небольшой эффект;
- комбинированный `improved_final` слегка уступает одиночному `h2_larger_img`;
- кастомная SSD ожидаемо уступает YOLOv11 (более простой backbone, скромнее аугментации, отсутствует предобучение всей модели на COCO);
- `custom_improved` относительно `custom_base` поднимает `mAP@50` (+0.0648) и `Recall` (+0.0970) ценой небольшого падения `Precision` (−0.0250) — типичный trade-off от добавления аугментации.
